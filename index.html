VeeCode Claude Gateway M2 pilot

Corporate gateway for Claude Code: Google identity in, attributable spend + audit logs out. Routes claude through https://claude-gateway.vee.codes.

Prerequisites

  • A Google Workspace account on a permitted domain
  • gcloud installed and authenticated once: gcloud auth login
  • Claude Code already installed (npm i -g @anthropic-ai/claude-code) — the installer locates it on PATH, it does not install it for you

Install (macOS / Linux)

curl -fsSL https://veecode-claude-gateway.github.io/install.sh | bash

Install (Windows PowerShell 7+)

iwr https://veecode-claude-gateway.github.io/install.ps1 | iex

What it does

Drops a thin claude wrapper ahead of the real binary on PATH. On invocation it exchanges your gcloud access token for a short-lived gateway virtual key (cached at ~/.config/claude-gateway/key for ~24h, mode 0600), points Claude Code at the gateway via ANTHROPIC_BASE_URL, then execs the real binary. You don't manage credentials.

The gateway logs metadata only — timestamp, model, token counts, your email. Conversation contents are not logged.

Verify

exec $SHELL    # or open a new terminal so PATH picks up the wrapper
claude --version

Troubleshooting

"gcloud not found" / "print-access-token failed"

Install the gcloud CLI and run gcloud auth login. The wrapper needs a live identity to mint a key.

"could not reach gateway"

Check VPN / network. The gateway is at https://claude-gateway.vee.codes.

Auth errors after working previously

The cached virtual key may have rotated. Force a refresh:

rm ~/.config/claude-gateway/key && claude --version

Switching gateways (advanced)

export CLAUDE_GATEWAY_URL=https://your-other-gateway.example
curl -fsSL https://veecode-claude-gateway.github.io/install.sh | bash

Reporting issues

File a GitHub issue at claude-gateway-parent/issues. Include the output of claude --version and a one-line description of what you were doing.
